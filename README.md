<div align="center"><h1>🎓 MTEC Schedule Bot</h1><p><b>Высоконагруженный асинхронный Telegram-бот для образовательного учреждения.</b></p></div>📌 О проектеMTEC_Bot — это комплексная информационная система для студентов и преподавателей колледжа. Бот парсит официальный сайт учреждения, мгновенно выдает расписание, генерирует удобные изображения-карточки занятий и интегрируется с электронным журналом.Проект спроектирован с упором на максимальную производительность: благодаря In-Memory кэшированию время ответа составляет ~0.001 сек, что полностью снимает нагрузку с целевого веб-сервера при массовых запросах пользователей.🚀 Ключевой функционалУмное расписание: Выдача расписания для студентов и преподавателей на сегодня/завтра/понедельник.Генерация изображений: Рендеринг расписания "на лету" в красивые PNG-карточки с помощью Pillow (выделение текущей пары, сетка, форматирование).Электронный журнал: Асинхронная авторизация на портале office.mtec.by, парсинг HTML с помощью BeautifulSoup4, очистка от лишних элементов и инъекция кастомного мобильного CSS.Поиск аудиторий: Отслеживание свободных кабинетов в режиме реального времени.Ролевая модель: Студент, Преподаватель, Куратор (с правом массовой рассылки своей группе), Администратор.Архив: Автоматическое сохранение сгенерированных расписаний и удобная навигация по прошедшим месяцам.⚙️ Техническая реализация и АрхитектураАсинхронность: Полностью асинхронный стек (aiogram 3, aiohttp, aiosqlite).Кэширование и Хэширование: Реализован глобальный словарь GLOBAL_SCHEDULE_CACHE. Для отслеживания изменений расписания используется вычисление MD5-хэшей данных.Background Tasks: Планировщик APScheduler отвечает за утренние рассылки (в 07:00) и фоновую проверку обновлений расписания каждые 5 минут.Anti-Spam Middleware: Написан кастомный BaseMiddleware для защиты от флуда. Алгоритм использует динамические накопительные штрафы (кулдауны), блокируя спамеров без падения бота.Оптимизация рассылок: Отправка графики оптимизирована: изображение рендерится и отправляется один раз, после чего его file_id кэшируется в памяти. Остальные пользователи получают фото по file_id без повторной загрузки и рендера.🛠 Установка и запуск (Local Setup)1.
