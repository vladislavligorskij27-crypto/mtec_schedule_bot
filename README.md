## **О проекте**

**MTEC\_Bot** — это комплексная информационная система для студентов и преподавателей колледжа. Бот парсит официальный сайт учреждения, мгновенно выдает расписание, генерирует удобные изображения-карточки занятий и интегрируется с электронным журналом.

Проект спроектирован с упором на максимальную производительность: благодаря In-Memory кэшированию время ответа составляет **\~0.001 сек**, что полностью снимает нагрузку с целевого веб\-сервера при массовых запросах пользователей.

## **Ключевой функционал**

* **Умное расписание:** Выдача расписания для студентов и преподавателей на сегодня/завтра/понедельник.  
* **Генерация изображений:** Рендеринг расписания "на лету" в красивые PNG-карточки с помощью Pillow (выделение текущей пары, сетка, форматирование).  
* **Электронный журнал:** Асинхронная авторизация на портале office.mtec.by, парсинг HTML с помощью BeautifulSoup4, очистка от лишних элементов и инъекция кастомного мобильного CSS.  
* **Поиск аудиторий:** Отслеживание свободных кабинетов в режиме реального времени.  
* **Ролевая модель:** Студент, Преподаватель, Куратор (с правом массовой рассылки своей группе), Администратор.  
* **Архив:** Автоматическое сохранение сгенерированных расписаний и удобная навигация по прошедшим месяцам.

## **Техническая реализация и Архитектура**

* **Асинхронность:** Полностью асинхронный стек (aiogram 3, aiohttp, aiosqlite).  
* **Кэширование и Хэширование:** Реализован глобальный словарь GLOBAL\_SCHEDULE\_CACHE. Для отслеживания изменений расписания используется вычисление **MD5-хэшей** данных.  
* **Background Tasks:** Планировщик APScheduler отвечает за утренние рассылки (в 07:00) и фоновую проверку обновлений расписания каждые 5 минут.  
* **Anti-Spam Middleware:** Написан кастомный BaseMiddleware для защиты от флуда. Алгоритм использует динамические накопительные штрафы (кулдауны), блокируя спамеров без падения бота.  
* **Оптимизация рассылок:** Отправка графики оптимизирована: изображение рендерится и отправляется один раз, после чего его file\_id кэшируется в памяти. Остальные пользователи получают фото по file\_id без повторной загрузки и рендера.

## **Установка и запуск (Local Setup)**

### **1\. Клонирование репозитория**

git clone \[https://github.com/vladislavligorskij27-crypto/mtec\_schedule\_bot.git\](https://github.com/vladislavligorskij27-crypto/mtec\_schedule\_bot.git)  
cd mtec\_schedule\_bot

### **2\. Создание виртуального окружения и установка зависимостей**

Рекомендуется использовать Python 3.11 или выше.

python \-m venv venv  
source venv/bin/activate  \# Для Linux/macOS  
venv\\Scripts\\activate     \# Для Windows

pip install \-r requirements.txt

*(Примечание: Убедитесь, что у вас установлен requirements.txt с aiogram, aiohttp, aiosqlite, beautifulsoup4, Pillow, pytz, python-dotenv, apscheduler).*

### **3\. Настройка переменных окружения**

Создайте файл .env в корневой папке проекта и добавьте туда свои данные:

BOT\_TOKEN=ваш\_токен\_от\_BotFather  
ADMIN\_ID=ваш\_telegram\_id

### **4\. Подготовка структуры данных**

Убедитесь, что в папке data/txt/ лежат файлы со списками:

* all\_groups.txt  
* all\_mentors.txt  
* rooms.txt

А в папке data/assets/ находятся нужные шрифты и картинки (Arial.ttf, logo.png, CALLS\_1.jpg, CALLS\_2.jpg).

### **5\. Запуск**

python main.py
